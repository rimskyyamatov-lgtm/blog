# L7 言語仕様書（暫定 v1.0）

## ■ 概要

本言語は、Brainfuck に類似した命令セットを持つが、
**プログラム長および実行状態により挙動が歪む決定的難解言語**である。

この言語のコンセプトは"Readable code, unreadable behavior."(読みやすいコード、動作は読みにくい。)

すべてのプログラムは同一入力に対して同一出力を返すが、
人間による直感的理解は困難である。

拡張子は`.l7`である

---

# ■ 命令セット

使用可能な命令は以下の8つのみ：

```txt
+ - > < [ ] . ,
```

各命令の**基本意味（理想状態）**：

| 命令  | 意味       |
| --- | -------- |
| `+` | 現在セルを +1 |
| `-` | 現在セルを -1 |
| `>` | ポインタを右へ  |
| `<` | ポインタを左へ  |
| `[` | ループ開始    |
| `]` | ループ終了    |
| `.` | 出力       |
| `,` | 入力       |

※ただし、実際の動作は以下の歪みにより変化する

---

# ■ メモリモデル

* 無限長配列（整数）
* 初期値はすべて 0
* ポインタ初期位置：0

---

# ■ 初期化

プログラム長を `L` とする

```txt
k = (L % 7) + 1
i = k % L
direction = +1
pointer = 0
```

---

# ■ 実行モデル

プログラムは以下のステップを繰り返す：

---

## ① 実行命令の決定（ズレ）

```txt
exec_index = (i + k + memory[pointer]) % L
cmd = code[exec_index]
```

---

## ② 命令の実行（歪み付き）

### ● 数値操作

```txt
+ → memory[pointer] += k
- → memory[pointer] -= k
```

---

### ● ポインタ操作

```txt
> → pointer += k
< → pointer -= k
```

---

### ● 入出力

```txt
. → (memory[pointer] + k) を出力
, → 入力値 XOR k を格納
```

---

### ● ループ（崩壊仕様）

#### 対応位置

```txt
対応する bracket = (現在の i + k) % L
```

#### 動作

* `[`：

  ```txt
  if memory[pointer] == 0:
      i = 対応する ]
  ```

* `]`：

  ```txt
  if memory[pointer] != 0:
      i = 対応する [
  ```

※ ネスト構造は保証されない

---

## ③ メモリ歪み

```txt
memory[pointer] ^= k
```

---

## ④ ポインタ自動移動

```txt
pointer += (memory[pointer] + k) % 2
```

---

## ⑤ 実行位置更新（非線形）

```txt
i += direction * ((k + memory[pointer]) % 3 + 1)
```

---

## ⑥ 実行方向反転

```txt
if ((i + k) % 5 == 0):
    direction *= -1
```

---

# ■ 終了条件

以下のいずれかで終了：

* `i` がプログラム範囲外に出た場合
* 実装依存の最大ステップ数に達した場合

---

# ■ 特徴

* 命令と実行位置が一致しない
* プログラム長により意味が変化
* メモリと実行が相互に影響
* ループ構造は非構造的
* 完全に決定的

---

# ■ 設計思想

* 最小の命令で最大の混乱を生む
* ランダム性を排除し、すべてを決定的にする
* 人間の直感を裏切るが、機械的には一貫性を保つ
